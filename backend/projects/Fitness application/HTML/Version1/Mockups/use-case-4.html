UC-004

Synchronize Data with Third-Party Health Platforms

A user connects their account to Apple Health or Google Fit to enable seamless data synchronization. Workout data and weight data are automatically exported to the user's central health hub.

Registered User (Beginner)
Priority: Medium
Page 4 of 5

Description

This use case enables users to seamlessly connect their fitness app data with established health platforms like Apple Health and Google Fit. By integrating with these ecosystems, users can maintain a centralized view of their health data across multiple applications. The synchronization happens automatically in the background, ensuring that workout metrics (calories burned, duration) and weight measurements are consistently updated across all connected platforms without requiring manual intervention.

Actors

Registered User (Beginner)

Preconditions

  • User has an active account with Apple Health or Google Fit.
  • User has the respective health platform app installed on their device.

Postconditions

  • The user's fitness data is bi-directionally synced with the connected health platform.
  • The app's status for the integration is updated to 'Active'.

Main Flow

Step-by-Step Process
1
User navigates to the 'Settings' or 'Profile' section of the app.
2
User finds and selects the 'Integrations' or 'Connected Apps' option.
3
User sees toggles for 'Apple Health' and 'Google Fit'.
4
User toggles the switch for their preferred platform (e.g., 'Apple Health').
5
The app prompts for authentication and redirects the user to the respective platform's permission screen.
6
The platform asks the user to grant the fitness app permission to read and/or write specific data types (e.g., Workouts, Weight).
7
User grants the requested permissions.
8
The user is redirected back to the fitness app, which now displays a 'Connected' status.
9
From this point on, relevant data is automatically synced in the background after each workout or when weight is logged.

Alternate Flows

User Revokes Permissions

If the user later revokes permissions in Apple Health/Google Fit settings, the app will detect this on the next sync attempt, disable the integration, and notify the user that data is no longer being synchronized.

Sync Failure

If a sync attempt fails due to a network error or API issue, the app will queue the data locally and retry the synchronization at a later time. It will not display technical errors to the end-user.

// Google Fit Integration
Fitness.getHistoryClient()
.readData(DataType.TYPE_WEIGHT)
.insertData(DataType.TYPE_WORKOUT)
