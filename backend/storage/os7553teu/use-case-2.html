UC-002 HIGH PRIORITY

Log Nutrition via Photo

User captures a photo of their meal, and the AI analyzes the food items and estimates nutritional information, reducing manual entry friction.

Actor

Beginner User

Preconditions

Camera & Login

Postconditions

Meal Logged

  1. 1

    User navigates to the Nutrition section and taps 'Add Meal'

  2. 2

    App opens camera interface with guidance for optimal photo capture

  3. 3

    User takes a photo of their meal

  4. 4

    App uploads photo to AI service (Google Cloud Vision/Clarifai)

  5. 5

    AI returns identified food items with confidence scores

  6. 6

    App displays recognized items and estimated nutritional information

  7. 7

    User confirms correct items or makes corrections

  8. 8

    User can adjust portion sizes if needed

  9. 9

    App saves the meal to the user's daily nutrition log

  10. 10

    App updates daily nutrition summary and progress towards goals

  • Low Confidence Handling

    If AI confidence is low, app prompts user to manually identify items

  • Meal Type Selection

    User can select meal type (breakfast, lunch, dinner, snack)

  • Additional Notes

    User can add notes about the meal

  • Error Recovery

    If photo analysis fails, user can retry or switch to manual entry

Key Differentiators

Reduces User Friction

Eliminates manual entry compared to MyFitnessPal

Leverages AI Trend

Modern approach using computer vision

Beginner-Friendly

Addresses desire for simplicity

Competitive Edge

Unique selling proposition in market
