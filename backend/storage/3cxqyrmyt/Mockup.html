UC-001

Log Workout Session

High Priority

Users can manually log their workout sessions including exercise type, duration, sets, reps, and weights. The system should save this data and make it available for progress tracking.

Actors

  • Registered User

Preconditions

  • User must be logged into the app
  • User must have completed a workout session

Main Flow

  1. User navigates to the 'Workout' section
  2. User selects 'Log New Workout' option
  3. User selects workout type
  4. User enters workout details
  5. User can add notes about the workout
  6. User saves the workout entry
  7. System validates and stores the workout data
  8. System confirms successful logging

Postconditions

  • Workout session is saved to user's profile
  • Data is available for progress charts
  • Calorie burn is calculated and added to daily total
UC-002

Track Daily Nutrition

High Priority

Users can log their daily food intake, track calories, macronutrients, and water consumption. The system should provide a food database and calculate nutritional values.

Actors

  • Registered User

Preconditions

  • User must be logged into the app
  • User must have food consumption data to log

Main Flow

  1. User navigates to the 'Nutrition' section
  2. User selects 'Add Food' option
  3. User searches for food item in the database
  4. User selects the correct food item
  5. User enters serving size and quantity
  6. System calculates nutritional values
  7. User confirms and saves the entry
  8. System updates daily nutrition totals

Alternate Flows

  • Scan barcode: User scans food item barcode using phone camera
  • Create custom food: User enters food name and nutritional information
UC-003

View Progress Charts

Medium Priority

Users can visualize their fitness progress through various charts and graphs showing workout history, weight changes, body measurements, and nutrition trends over time.

Actors

  • Registered User

Preconditions

  • User must be logged into the app
  • User must have logged at least one workout or nutrition entry

Main Flow

  1. User navigates to the 'Progress' section
  2. User selects the type of chart to view
  3. User selects the time period
  4. System generates and displays the relevant chart
  5. User can interact with chart to view detailed data points
  6. User can switch between different chart types
  7. User can export or share the chart if desired

Business Value

Provides visual feedback and motivation to users, helping them stay committed to their fitness journey

UC-004

Connect Wearable Device

High Priority

Users can connect their wearable devices (Apple Watch, Fitbit) to automatically sync workout data, heart rate, steps, and other health metrics with the app.

Actors

  • Registered User

Preconditions

  • User must be logged into the app
  • User must have a compatible wearable device
  • User must have the wearable's companion app installed

Main Flow

  1. User navigates to 'Settings' > 'Connected Devices'
  2. User selects 'Add New Device'
  3. System displays list of supported devices
  4. User selects their device type
  5. System initiates OAuth/authentication flow
  6. User grants permission on the wearable's platform
  7. System confirms successful connection
  8. System begins syncing historical data

Postconditions

  • Wearable device is connected to the app
  • Data sync is established
  • User can view synced data in the app

UC-006

Sync Data from Wearable

High Priority

The system automatically or manually syncs fitness data from connected wearable devices, ensuring the user's fitness information is up-to-date across all platforms.

Actors

  • System
  • Registered User

Preconditions

  • User must have at least one wearable device connected
  • Wearable must have new data to sync
  • App must have internet connectivity

Main Flow

  1. System checks for new data from connected wearables
  2. System retrieves workout data, steps, heart rate, sleep data
  3. System processes and normalizes the data
  4. System updates user's fitness profile
  5. System recalculates progress metrics and goals
  6. System sends notification to user about synced data
  7. User can view the newly synced data

Alternate Flows

  • Manual sync: User initiates manual sync from settings
  • Sync conflict resolution: System detects conflicting data entries and applies resolution rules
