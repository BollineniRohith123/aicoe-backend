UC-004

Filter Tasks by Status

A user views a subset of their tasks based on their completion status (All, Active, Completed). This enhances usability as the list grows, a standard feature in competing applications.

Actors

User

Preconditions

  • The user has at least one task in their list.

Priority & Value

Priority: Medium

Improves user experience by allowing efficient navigation through large task lists, reducing cognitive load and helping users focus on relevant tasks.

Main Flow

  1. The user views the task filter controls, typically buttons or links labeled 'All', 'Active', and 'Completed'.
  2. The user clicks on one of the filter options.
  3. The frontend updates the filter state in the application state management.
  4. The UI re-renders the task list, showing only tasks that match the selected filter criteria.
  5. The selected filter button is visually highlighted to indicate the active filter.

Alternate Flows

No Tasks Match Filter

  1. The user selects a filter that has no matching tasks.
  2. The UI displays an empty state message (e.g., 'No active tasks').
  3. The filter button remains highlighted to show the current selection.

Postconditions

  • The task list is filtered according to the selected status.
  • The UI displays only the tasks matching the filter criteria.
  • The filter state is maintained in the application state.

Business Value

This feature significantly enhances the user experience by enabling efficient navigation through potentially large task lists. As the number of tasks grows, users need ways to focus on what's relevant at any given time. The filtering capability reduces cognitive load, improves productivity, and aligns with user expectations from modern task management applications. This feature is essential for maintaining usability as the application scales.
