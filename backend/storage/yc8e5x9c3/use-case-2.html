UC-002

Mark Task as Complete

High Priority
A user changes the status of a task from 'active' to 'completed'. This action provides immediate visual feedback and updates the task's status in the backend, adhering to accessibility standards.

Actors

  • User

Preconditions

  • The user has at least one active task in their list.
  • The task list is visible on the screen.

Postconditions

  • The task's status is updated in the UI.
  • The task's status is persisted in the backend database.
  • If a filter is active (e.g., 'Active Tasks'), the task may disappear from the current view.
The user views a task in the list (rendered by a TaskItem component).
The user clicks the checkbox associated with the task.
The system updates the task's status in the frontend state to 'completed'.
The UI immediately updates: the task text receives a strikethrough style, and the checkbox is checked.
The system sends an asynchronous PATCH request to the backend endpoint (e.g., PATCH /tasks/:id) with a JSON payload like { "status": "completed" }.
The backend updates the task's status in the data store and responds with a success status.
Un-marking a Task
The user clicks the checkbox of a completed task.
The system reverts the task's status to 'active'.
Visual strikethrough is removed.
A PATCH request is sent to update the status to 'active'.

Business Value

Enables users to track their progress, which is the primary benefit of a to-do list. This feature is a baseline expectation set by market leaders like Todoist and Microsoft To Do.
