UC-001

User Onboards and Sets Up Profile

A new user downloads the app and completes the onboarding process to create a personalized profile. This process is designed to be simple and welcoming for fitness beginners, avoiding overwhelming information and setting them up for success with clear initial goals.

New User (Fitness Beginner)
High Priority
Created Today
Actors
  • New User (Fitness Beginner)
Preconditions
  • User has downloaded and installed the mobile application.
Postconditions
  • User account is created.
  • User profile is configured with initial goals and preferences.
  • Third-party health integrations are enabled (if consent was given).
  • User is on the main dashboard and ready to use the app.

Flow Diagrams

Main Flow
1
User opens the app for the first time.
2
App presents a welcome screen highlighting its core value: simple fitness and nutrition tracking for beginners.
3
User taps 'Get Started'.
4
App prompts for account creation (email/password or social login).
5
User creates an account.
6
App requests basic profile information: name, age, gender, and fitness level (e.g., 'Just Starting', 'Some Experience').
7
User provides the requested information.
8
App presents primary fitness goals in a simple, multiple-choice format (e.g., 'Lose Weight', 'Build Muscle', 'Get Fitter').
9
User selects one or more goals.
10
App requests permission to integrate with Apple Health or Google Fit, explaining the benefit of automatic data syncing.
11
User grants or denies permission.
12
App presents a brief, interactive tutorial showcasing the three main features: 'Log a Workout', 'Track a Meal', and 'View Progress'.
13
User completes the tutorial.
14
App navigates the user to the main dashboard, pre-populated with a recommended 'Beginner' workout plan.
Alternate Flows

Condition: User declines integration with Apple Health/Google Fit

The app notes the preference and proceeds. The user can enable integration later in settings. Manual entry will be the primary method.

Condition: User exits the onboarding process prematurely

The app saves the progress made so far. Upon reopening, the app prompts the user to complete the setup from where they left off.

Technical Details

Security Requirements

  • • Password encryption
  • • OAuth 2.0 for social login
  • • GDPR compliance
  • • Health data encryption

Performance Metrics

  • • Onboarding completion: <3 minutes
  • • Load time: <2 seconds
  • • Drop-off rate: <15%
  • • API response: <500ms

Platform Support

  • • iOS 14.0+
  • • Android 8.0+
  • • Responsive web view
  • • Progressive Web App

Data Storage

  • • User profiles in PostgreSQL
  • • Session data in Redis
  • • Analytics in BigQuery
  • • Backups daily
Business Value

A smooth onboarding process is critical for user retention, especially for beginners who may be intimidated. This use case directly addresses the pain point of users feeling overwhelmed by complex apps and establishes the foundation for a personalized experience, a key industry trend.
