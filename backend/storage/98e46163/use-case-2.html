UC-002

User Logs a Workout Session

High Priority

The user selects a pre-made workout plan or a custom exercise and records their performance (sets, reps, weight) for each exercise during a session. The interface is designed to be minimal and distraction-free, focusing on the immediate task at hand.

Actors

  • Primary User (Fitness Beginner)

Business Value

This is a core functional requirement that delivers the app's primary value proposition. It directly addresses the need for simple workout logging and provides the essential data for progress visualization, a key user expectation for motivation.

Key Features

  • Minimal, distraction-free interface
  • Pre-made and custom workout support
  • Visual exercise demonstrations
  • Real-time progress tracking
  • Health platform integration

Main Flow Steps

1
User navigates to the 'Workout' tab from the main dashboard.
2
User views their current workout plan for the day and taps 'Start Workout'.
3
App displays the first exercise in the plan, including its name, target sets/reps, and a visual demonstration (image or GIF).
4
User performs the first set of the exercise.
5
User taps the 'Log Set' button.
6
App presents input fields for 'Reps' and 'Weight' (if applicable).
7
User enters the values and taps 'Save'.
8
App records the set, updates progress for the exercise, and displays a brief success animation.
9
App automatically presents the next set of the same exercise or the next exercise in the plan.
10
User repeats steps 4-8 until all exercises and sets are completed.
11
Upon completion, the app displays a 'Workout Complete' summary screen showing total exercises, sets, and an estimated calorie burn.
12
User taps 'Finish'.
13
The workout data is saved to the user's history and synced with Apple Health/Google Fit (if enabled).

Alternate Flow Scenarios

Condition: User needs to see a better demonstration of an exercise

The user taps a 'Watch Video' link, which opens a trusted third-party video (e.g., YouTube) in the device's browser or in-app player, adhering to the decision to avoid hosting video.

Condition: User cannot complete an exercise or needs to skip it

The user taps a 'Skip Exercise' button. The app logs the skipped exercise and allows the user to proceed to the next one.

Condition: User needs to pause the workout

The user taps a 'Pause' button. The app freezes the timer and current state. The user can tap 'Resume' to continue.

Preconditions

  • User has completed onboarding.
  • User has selected a workout plan or has access to the exercise library.

Postconditions

  • All workout data (exercises, sets, reps, weight, duration) is saved to the user's profile.
  • User's progress visualizations are updated.
  • Data is synced with integrated third-party health platforms.

Technical Requirements

  • Real-time data synchronization
  • Apple Health/Google Fit integration
  • Offline workout logging capability
  • Visual exercise demonstrations (images/GIFs)
  • Progress tracking and analytics

Strategic Impact

This use case represents the core functionality that differentiates our fitness app in the market. By providing a seamless, distraction-free workout logging experience, we address the primary pain point of fitness enthusiasts who want to track their progress without interrupting their flow. The integration with popular health platforms ensures our app becomes part of the user's broader fitness ecosystem, increasing retention and user lifetime value.
