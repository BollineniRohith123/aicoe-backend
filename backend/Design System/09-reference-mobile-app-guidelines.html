Mobile Pattern Guidelines

  • Swipe Gestures: Use for navigation between similar content (carousel, tabs)
  • Pull to Refresh: Standard gesture for updating content in lists/feeds
  • Action Sheets: Present contextual actions from bottom of screen
  • Touch Feedback: Provide immediate visual feedback on all interactive elements
  • Scroll Behavior: Use momentum scrolling and snap points for better UX
  • Gesture Conflicts: Avoid conflicting gestures (e.g., swipe for both delete and navigate)

Mobile Development Guidelines

Touch Target Sizes

Minimum Touch Target: 44×44 pixels (11mm)

This ensures comfortable interaction for users of all abilities and thumb sizes. Smaller targets lead to frustration and errors.

DO

Adequate touch target with comfortable spacing

DON'T

Touch target too small, difficult to tap accurately

/* Minimum touch targets */ .mobile-button { min-height: 44px; min-width: 44px; padding: 12px 24px; } /* Spacing between touch targets */ .mobile-button + .mobile-button { margin-top: 8px; /* Minimum 8px spacing */ }

Mobile Typography

Minimum Font Size: 16px for body text

Prevents zoom on iOS Safari and ensures readability. Use larger sizes for primary content.

Safe Areas & Notches

Respect Device Safe Areas

Modern devices have notches, rounded corners, and home indicators. Use safe area insets to avoid content being obscured.

/* Safe area insets for iOS */ .mobile-header { padding-top: env(safe-area-inset-top); padding-left: env(safe-area-inset-left); padding-right: env(safe-area-inset-right); } .mobile-footer { padding-bottom: env(safe-area-inset-bottom); } /* Viewport meta tag */ <meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
Note: Always test on physical devices with notches (iPhone X+, modern Android) to ensure proper spacing

Performance Optimization

Image Optimization

  • Use WebP format with JPG/PNG fallback
  • Implement lazy loading for images below fold
  • Provide 1x, 2x, and 3x resolutions for retina displays
  • Compress images before deployment (aim for <200KB)
  • Use srcset and sizes attributes for responsive images

JavaScript Best Practices

  • Code splitting: Load only what's needed for current view
  • Minimize third-party libraries (every KB counts on mobile)
  • Use passive event listeners for scroll/touch handlers
  • Debounce expensive operations (search, calculations)
  • Leverage service workers for offline capability

Animation Performance

/* Use transform and opacity for 60fps animations */ .smooth-animation { /* ✓ Good - GPU accelerated */ transform: translateX(100px); opacity: 0.5; will-change: transform, opacity; } /* ✗ Avoid - causes reflow/repaint */ .janky-animation { left: 100px; /* Don't animate position properties */ width: 200px; /* Don't animate layout properties */ } /* Reduce motion for accessibility */ @media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }

Offline Support & Network Resilience

Design for Unreliable Networks

Mobile networks are often slow or intermittent. Build resilience into your application.

Strategies

Local Caching

Cache API responses, images, and assets using service workers or IndexedDB

Retry Logic

Implement exponential backoff for failed requests with user-friendly error messages

Optimistic Updates

Update UI immediately, sync with server in background. Rollback if needed.

Offline Indicators

Clearly communicate connection status and available offline features

Mobile Input Methods

Optimize Keyboard Experience

Use appropriate input types to show the right keyboard layout and validation.

<input type="email" inputmode="email" autocomplete="email"> <input type="tel" inputmode="tel" autocomplete="tel"> <input type="url" inputmode="url" autocomplete="url"> <input type="number" inputmode="numeric" pattern="[0-9]*"> <input type="search" inputmode="search" autocomplete="off"> <input type="date">
Tip: Use autocomplete attributes to enable autofill and password managers

Platform-Specific Guidelines

iOS Guidelines

Navigation

  • Back button top-left with chevron
  • Tab bar at bottom (max 5 tabs)
  • Swipe from left edge to go back
  • Navigation bar: 44px height

Interactions

  • Swipe to delete in lists
  • Pull to refresh for content updates
  • Action sheets from bottom
  • Haptic feedback for important actions

Design Elements

  • San Francisco font system
  • Translucent backgrounds (blur effects)
  • Large titles in navigation bar
  • System blue for primary actions (#007AFF)
Reference: Apple Human Interface Guidelines

Android Guidelines

Navigation

  • Back button in system nav (bottom/gesture)
  • Bottom nav bar or drawer menu
  • FAB (Floating Action Button) for primary action
  • App bar: 56dp height

Interactions

  • Swipe to refresh (circular progress)
  • Bottom sheets for actions
  • Snackbars for temporary messages
  • Long-press for contextual menus

Design Elements

  • Roboto font system
  • Material elevation (shadows)
  • Ripple effects on touch
  • 8dp grid system
Reference: Material Design 3
